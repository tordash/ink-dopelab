## คู่แข่งถูกกว่าทุกตัว... จริงหรือ?

ลูกค้า e-commerce ส่งข้อความมาตอนเที่ยง:

"พี่ ดูราคาคู่แข่งหน่อย **เขาถูกกว่าเราเกือบทุกตัว** ทำไงดี"

ร้านขายสินค้าออนไลน์ 500+ รายการ คู่แข่งหลายเจ้า ราคาเปลี่ยนทุกวัน

เช็คมือ? เทียบทีละตัว x หลายร้าน = เป็นไปไม่ได้

เลยสร้างระบบ AI ขึ้นมาแก้ปัญหานี้ — และสิ่งที่พบทำให้ทุกอย่างเปลี่ยน

---

## สิ่งที่ AI ทำใน 20 วินาที

```
1. Scrape สินค้าร้านลูกค้า (175 รายการ)
2. Scrape สินค้าคู่แข่ง (468 รายการ) ผ่าน API
3. จับคู่สินค้าอัตโนมัติ — แบรนด์ + รุ่น + ประเภท
4. เทียบราคาทุกคู่
5. ★ เช็คสต็อค — สินค้ามีจริงหรือแค่ตั้งราคาไว้?
6. เขียนรายงานพร้อม highlight สี
```

ขั้นตอนที่ 5 คือตัวเปลี่ยนเกม

---

## Plot Twist — จับโกงได้

ผลเทียบราคารอบแรก: คู่แข่งถูกกว่า **29 รายการ** จาก 50 ที่จับคู่ได้

ตกใจ — เกินครึ่ง!

แต่พอเช็ค stock ของ 29 รายการนั้น:

**สินค้าที่คู่แข่ง "ถูกกว่า" 14 รายการ → หมด stock ทุกตัว**

ทดสอบซ้ำ — กดซื้อจริง → "สินค้าหมด" ทุกตัว

เป็นกลยุทธ์: **ตั้งราคาถูกกับของที่ไม่มีขาย** ให้ดูดีใน price comparison แต่ซื้อไม่ได้จริง

---

## Red / Green / Grey Framework

สร้าง framework ง่ายๆ ให้ลูกค้าอ่านผล:

| สถานะ | สี | จำนวน | ความหมาย |
|--------|-----|--------|-----------|
| **ภัยจริง** (ถูกกว่า + มีของ) | แดง | 15 | ต้องปรับราคา |
| **ปั่นราคา** (ถูกกว่า + หมดของ) | เทา | 14 | ไม่ต้องกังวล |
| **เราถูกกว่า** | เขียว | 13 | ปลอดภัย |
| **ราคาเท่ากัน** | ขาว | 8 | OK |

**ก่อน**: คิดว่าแพ้ 29 รายการ → panic
**หลัง**: แพ้จริงแค่ 15 รายการ → **ลดแพนิค 48%**

และ 15 รายการที่เป็นภัยจริง ก็ปรับราคาได้ตรงจุด — ไม่ต้องลดทั้งร้าน

---

## ทำไมเรื่องนี้สำคัญกับทุก E-commerce

### 1. ข้อมูลราคาอย่างเดียวไม่พอ

ทุก price comparison tool เทียบแค่ราคา ไม่เช็ค stock

ผลลัพธ์: ตกใจกับ "คู่แข่งถูกกว่า" ที่ซื้อไม่ได้จริง → ลดราคาเปล่าๆ → กำไรหาย

### 2. การตรวจสอบมือ scale ไม่ได้

500 SKU x 5 คู่แข่ง = 2,500 คู่เทียบ

ถ้าเช็คมือ ใช้เวลา 1 นาทีต่อคู่ = **41 ชั่วโมง** ต่อรอบ

AI ทำเสร็จใน **20 วินาที**

### 3. คู่แข่งใช้กลยุทธ์ราคาที่ซับซ้อนขึ้น

ปั่นราคาสินค้าหมด stock ไม่ใช่เรื่องผิดกฎหมาย — แต่ถ้าไม่มีเครื่องมือตรวจจับ จะเสียเปรียบทุกครั้ง

## เบื้องหลังระบบ

ทั้งหมดสร้างบน Claude Code + Python:

**Data Collection:**
- Scrape ด้วย API ของ platform (ไม่ใช่ web scraping ที่ผิด TOS)
- Product matching ด้วย brand + model + category fuzzy match
- Stock check ด้วย availability API

**Analysis:**
- Price comparison matrix
- Stock validation layer (ตัวที่ทำให้ทุกอย่างต่าง)
- Red/Green/Grey classification อัตโนมัติ

**Output:**
- Report พร้อม action items ที่ชัดเจน
- "ปรับราคา 15 ตัวนี้" vs "ไม่ต้องทำอะไรกับ 14 ตัวนี้"
- ประหยัดเวลาตัดสินใจจากหลายวันเหลือ 20 วินาที

---

## ตัวเลขสรุป

| Metric | ค่า |
|--------|-----|
| SKU ที่เช็ค | 175 (ลูกค้า) vs 468 (คู่แข่ง) |
| จับคู่ได้ | 50 คู่ |
| "ภัยจริง" ที่ต้อง action | 15 (ไม่ใช่ 29) |
| ลดแพนิค | **-48%** |
| เวลาเทียบมือ | ~41 ชั่วโมง |
| เวลา AI | **20 วินาที** |
| เวลาสร้างระบบ | ~3 ชั่วโมง (Claude Code) |

## สิ่งที่ได้เรียนรู้

**1. Layer ที่ทำให้ต่าง = stock check**

ทุกคนเทียบราคาได้ แต่คนที่เช็ค stock ด้วยจะเห็นภาพจริง ไม่ตกใจเปล่า

**2. AI ไม่ได้แทนคน — แต่ให้ข้อมูลที่ดีกว่า**

ลูกค้ายังตัดสินใจเองว่าจะปรับราคาตัวไหน AI แค่บอกว่า "15 ตัวนี้เป็นภัยจริง ที่เหลือไม่ต้อง panic"

**3. ระบบที่สร้างใน 3 ชั่วโมง ใช้ได้ทุกสัปดาห์**

สร้างครั้งเดียว รันซ้ำได้ตลอด ข้อมูลอัปเดตทุกรอบ

---

## เริ่มต้นยังไง (ไม่ต้อง code เอง)

ถ้าอยากเริ่มเช็คราคาคู่แข่ง:

**Level 1: Manual sampling (0 บาท)**
เลือก 10 SKU สำคัญที่สุด เทียบมือทุกสัปดาห์ จดลง spreadsheet

**Level 2: Google Alerts + Sheet (0 บาท)**
ตั้ง Google Alerts ชื่อสินค้า + ชื่อคู่แข่ง เก็บข่าวราคาเปลี่ยนอัตโนมัติ

**Level 3: AI automation (เหมือนในบทความ)**
Claude Code + Python scraper + stock validation ทำครั้งเดียว ใช้ได้ตลอด

## Prompt จริงที่ใช้

```
# Prompt สำหรับสร้าง scraper
สร้างระบบเทียบราคาสินค้าลูกค้า vs คู่แข่ง
scrape ผ่าน API, fuzzy match brand+model,
เช็ค stock availability, classify Red/Green/Grey,
output เป็น report พร้อม action items

# Prompt สำหรับ report
วิเคราะห์ผลเทียบราคา — แยก "ภัยจริง" (ถูกกว่า+มีของ) 
ออกจาก "ปั่นราคา" (ถูกกว่า+หมดของ) ให้ชัด
สรุปว่าต้อง action กี่ตัว ไม่ต้อง panic กี่ตัว
```
