## ก่อนหน้านี้ — Excel กับความหวัง

ร้านของผมขายเนื้อ วัตถุดิบหลักคือเนื้อสดหลายชนิด — สันนอก, สันใน, เสือร้องไห้, คอหมู, หมูสามชั้น และอีกหลายสิบรายการ

ก่อนหน้านี้ใช้ **Excel** ธรรมดา พนักงานรับของมา ก็จดลงตาราง... ถ้าจำได้ ถ้าว่าง ถ้าไม่ลืม

ปัญหาคือ:
- ❌ ข้อมูลไม่ real-time — กว่าจะรู้สต็อคจริง ต้องรอคนอัพเดท
- ❌ ตรวจสอบย้อนหลังไม่ได้ — ของหาย ไม่รู้หายตอนไหน
- ❌ วิเคราะห์ไม่ได้ — Excel แค่ตาราง ไม่มี query, ไม่มี dashboard

## สิ่งที่ให้ AI ช่วยสร้าง

ผมไม่ได้เขียนโค้ดเป็น แค่อธิบายให้ AI ฟังว่า:
- ร้านรับวัตถุดิบอะไรบ้าง
- ขั้นตอนรับของทำยังไง
- อยากรู้อะไรจากข้อมูล

AI ก็ช่วยสร้างระบบให้:

### ชั้นที่ 1: Airtable — หน้าบ้านที่พนักงานใช้

Airtable เป็นเหมือน Excel ที่ฉลาดกว่า:
- พนักงานกรอกข้อมูลผ่านฟอร์มบนมือถือ
- ทุกรายการมี **Bag ID** อัตโนมัติ (เช่น `BEEF000123`)
- เชื่อมตารางวัตถุดิบ → ดึงหน่วย, หมวดหมู่, ราคาต่อ kg อัตโนมัติ
- Gallery view สำหรับดูสต็อคแบบ visual

### ชั้นที่ 2: SQLite — หลังบ้านที่ AI วิเคราะห์

ข้อมูลจาก Airtable ถูก sync มาเก็บใน SQLite database เพื่อให้ AI วิเคราะห์ได้:

```sql
-- ดูยอดรับเข้าแยกตามวัตถุดิบ ย้อนหลัง 7 วัน
SELECT
  ingredient_name,
  COUNT(*) as bags,
  ROUND(SUM(weight_kg), 1) as total_kg
FROM inventory_movements
WHERE type = 'receive'
  AND date >= date('now', '-7 days')
GROUP BY ingredient_name
ORDER BY total_kg DESC;
```

<Callout variant="info" title="ทำไมต้อง 2 ชั้น?">
Airtable เหมาะกับพนักงาน (UI ง่าย, มือถือได้) แต่ช้าสำหรับ query ข้อมูลเยอะๆ SQLite เร็วมากสำหรับวิเคราะห์ข้อมูล แต่พนักงานใช้ไม่ได้ เลยใช้คู่กัน
</Callout>

## ตัวเลขจริง

หลังจากใช้ระบบมา 1 เดือน:

| ตัวชี้วัด | ก่อน (Excel) | หลัง (Airtable+SQLite) |
|-----------|-------------|----------------------|
| วัตถุดิบที่บันทึก | ~50 รายการ/เดือน | **757 ถุง/เดือน** |
| เวลาบันทึกต่อรายการ | 3-5 นาที | **30 วินาที** |
| ตรวจสอบย้อนหลัง | ไม่ได้ | **ทุกรายการ มี Bag ID** |
| รายงานสรุป | ทำมือ | **AI สร้างให้อัตโนมัติ** |
| ข้อมูลหาย | บ่อย | **0 ครั้ง** |

## สิ่งที่เรียนรู้

### 1. ระบบต้องง่ายกว่าของเดิม
ถ้าระบบใหม่ยากกว่า Excel → พนักงานจะไม่ใช้ Airtable form บนมือถือง่ายกว่า Excel มาก ผลคือพนักงานบันทึกทุกรายการจริงๆ

### 2. Bag ID เปลี่ยนทุกอย่าง
พอทุกถุงมี ID เฉพาะ สิ่งที่เคยเป็นไปไม่ได้ก็ทำได้:
- ติดตามว่าถุงไหนส่งไปสาขาไหน
- รู้ว่าถุงไหนค้างนานเกิน
- ตรวจสอบย้อนหลังเมื่อมีปัญหา

### 3. ไม่ต้องเขียนโค้ดเป็นก็ได้
ผมอธิบายให้ AI ฟังเป็นภาษาคน AI สร้าง database schema, เขียน import script, ทำ query สำเร็จรูปให้ — ผมแค่บอกว่าอยากรู้อะไร

<Callout variant="success" title="Key Takeaway">
เทคโนโลยีไม่ใช่ปัญหา ปัญหาคือเราอธิบายให้ AI เข้าใจธุรกิจเราได้ดีแค่ไหน ยิ่งอธิบายละเอียด → ระบบยิ่งตรงจุด
</Callout>

## อยากลองทำบ้าง?

ขั้นตอนหลักๆ:
1. **จด process ร้านคุณ** — รับของยังไง เก็บยังไง เบิกยังไง
2. **เลือก front-end** — Airtable, Google Forms, หรือ Notion
3. **ให้ AI ช่วยสร้าง database** — อธิบายให้ฟังเป็นภาษาคน
4. **เริ่มจากส่วนเล็กๆ** — อย่าทำทั้งระบบทีเดียว เริ่มจากบันทึกรับเข้าก่อน

ระบบไม่ต้องสมบูรณ์แบบตั้งแต่วันแรก แค่เริ่มดีกว่าไม่เริ่ม
