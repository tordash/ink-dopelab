## Head of Design ที่ Claude พูดเองว่า "Design Process ตายแล้ว"

![AI ฆ่า Design Process — จาก Figma สู่ MCP](/images/blog/design-dead.jpg)

ไม่ใช่คนนอกพูด ไม่ใช่ YouTuber ปั่น ไม่ใช่ Twitter hot take

**Jenny Wen** — Head of Design ที่ Anthropic (บริษัทที่สร้าง Claude) อดีตทีม Figma โดยตรง — พูดเปิดเผยว่า **"The design process is dead"**

คนที่เคยสร้าง Figma เอง มาบอกว่า design process แบบเดิมไม่ work อีกแล้ว

สัญญาณนี้หนักกว่าที่หลายคนคิด

## Figma + Anthropic ปล่อย MCP Integration — Bidirectional

วันที่ 17 กุมภาพันธ์ 2026 Figma กับ Anthropic ประกาศ MCP (Model Context Protocol) integration แบบ **bidirectional** — ไม่ใช่แค่ AI อ่าน Figma ได้ แต่ **AI เขียนกลับเข้า Figma ได้ด้วย**

ก่อนหน้านี้ AI กับ design tools มันแยกขาดจากกัน Designer ออกแบบใน Figma → ส่ง spec ให้ developer → developer ไปเขียน code ใหม่ทั้งหมด

ตอนนี้ Claude Code สามารถ:

- **อ่าน Figma file** — ดู component, layout, spacing, typography ทั้งหมด
- **แปลงเป็น code** — สร้าง HTML/CSS/React ตรงจาก design ที่อยู่ใน Figma
- **เขียนกลับ** — สร้าง component ใหม่ แก้ไข design ใน Figma จากคำสั่ง text

พูดง่ายๆ คือ ขั้นตอน design → handoff → code ที่เคยใช้เวลาเป็นวัน ตอนนี้ **ยุบเหลือไม่กี่นาที**

## มีคนสร้าง "Design Studio" Plugin สำหรับ Claude Code แล้ว

สิ่งที่น่าสนใจที่สุดคือไม่ใช่แค่คนพูดเรื่องนี้ — มีคนลงมือสร้างระบบจริง

มี developer สร้าง **"Design Studio" plugin** สำหรับ Claude Code ที่มี:

- **9 Design Roles** — UX Researcher, UI Designer, Interaction Designer, Visual Designer, Design System Lead ฯลฯ
- **16 Commands** — ตั้งแต่ generate wireframe, create component, audit accessibility ไปจนถึง export production assets
- **5 AI Agents** — ทำงานประสานกัน เหมือนมี design team ย่อส่วน

แต่ละ role มี expertise เฉพาะทาง — เวลาสั่งงาน ระบบจะ route ไปหา agent ที่เหมาะสมเอง

ฟังดูเหมือน sci-fi แต่มันทำงานจริง และ code เป็น open source ให้ลองใช้ได้



## แล้ว Designer ต้องปรับตัวยังไง?

ผมต้องพูดตรงๆ ก่อนว่า — **ผมไม่ใช่ designer** ผมเป็นคนทำ agency ที่ใช้ AI tools ทำงานทุกวัน และสิ่งที่ผมเห็นจากฝั่ง production คือ

### สิ่งที่ AI ทำได้ดีแล้ววันนี้

- **Layout และ component** — Claude Code สร้าง responsive UI ที่ใช้งานได้จริง ไม่ใช่แค่ mockup
- **Design system** — สร้าง color palette, typography scale, spacing system ที่ consistent
- **Iteration** — เปลี่ยน design ทั้งหมดได้ใน 30 วินาที ไม่ต้องลาก pixel ใหม่

### สิ่งที่ AI ยังทำไม่ได้ (ตอนนี้)

- **เข้าใจ brand essence** — AI ทำ pattern ได้ แต่ยังไม่เข้าใจว่าทำไม brand นี้ต้อง "รู้สึก" แบบนี้
- **ตัดสินใจเรื่อง trade-off** — เลือกระหว่าง "สวยกว่า" vs "ใช้ง่ายกว่า" ยังต้องใช้คน
- **วิจัย user จริง** — AI สร้าง persona ได้ แต่นั่งฟัง user บ่นไม่ได้

ผมคิดว่า designer ที่จะอยู่รอดคือคนที่เปลี่ยนจาก **"คนลาก pixel"** มาเป็น **"คนตัดสินใจว่า pixel ไหนควรอยู่ตรงไหน แล้วสั่ง AI ทำ"**

## Workflow ที่เปลี่ยนไป (แต่ผมจะไม่โชว์ทั้งหมด)

ขั้นตอนคร่าวๆ ที่เราใช้:

1. **Brief** — เขียน prompt อธิบายสิ่งที่ต้องการ (ภาษาคน ไม่ใช่ design spec)
2. **Generate** — Claude Code สร้าง HTML/CSS ตรง ไม่ผ่าน wireframe
3. **Review** — ดู output, ปรับ prompt ถ้าไม่ตรง
4. **Iterate** — สั่ง "เปลี่ยนสีเป็น...", "ย้าย section นี้ขึ้น...", "เพิ่ม animation..."
5. **Deploy** — push ขึ้น production ได้เลย



ขั้นตอนที่ 3-4 คือหัวใจ — **AI ทำงาน draft แรก, คนทำงาน direction** ไม่ใช่คนทำทุกอย่างเอง

แต่รายละเอียดลึกๆ เรื่อง prompt engineering สำหรับ design, วิธี setup MCP, และ workflow เต็มๆ — ไว้ผมเขียนแยกอีกบทความ เพราะมีเนื้อหาเยอะเกินจะยัดลงที่นี่ได้หมด

## สรุป — Design Process ไม่ได้ตาย แต่กำลังกลายพันธุ์

ผมไม่คิดว่า designer จะตกงานหมดพรุ่งนี้ แต่ผมคิดว่า **role ของ designer กำลังเปลี่ยน** อย่างรวดเร็วมาก

สิ่งที่เกิดขึ้น:

- **Jenny Wen** (อดีต Figma, ปัจจุบัน Head of Design ที่ Claude) พูดเองว่า process เดิมตายแล้ว
- **Figma + Anthropic** ปล่อย MCP integration ที่ AI อ่าน-เขียน Figma ได้ bidirectional
- **Community** เริ่มตั้งคำถามว่า Figma ยังจำเป็นมั้ย
- **มีคนสร้าง Design Studio** plugin ที่ทำงานแทน design team ได้จริง
- **Agency อย่างเรา** ใช้ Claude Code ออกแบบงานจริงทุกวัน โดยไม่เปิด Figma

สัญญาณชัดเจนมากว่า designer ที่ยังทำงานแบบเดิม 100% จะถูก disrupt ภายใน 1-2 ปี

แต่ designer ที่เรียนรู้วิธีใช้ AI เป็นเครื่องมือ — ใช้ Claude Code สร้าง prototype, ใช้ MCP เชื่อม Figma, ใช้ AI iterate เร็วขึ้น 10 เท่า — คนเหล่านั้นจะมี value สูงขึ้นกว่าเดิมมาก

**เครื่องมือเปลี่ยน แต่คนที่ตัดสินใจเรื่อง design ยังจำเป็น** — แค่ต้องเปลี่ยนวิธีทำงาน
