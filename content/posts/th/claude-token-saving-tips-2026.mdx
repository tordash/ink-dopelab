จ่าย Claude Pro เดือนละ $20 แต่ใช้ไปไม่ถึงครึ่งวัน token ก็หมด — ถ้าเป็นคุณ ไม่ได้อยู่คนเดียว

โพสต์ของ Thedigitalkinggg บน Facebook ที่สรุป 10 วิธีประหยัด token ถูกแชร์กว่า 236 ครั้ง กดไลค์ 545 ครั้ง ภายในวันเดียว — พิสูจน์ว่านี่คือปัญหาที่คนใช้ Claude เจอกันทั่วหน้า

เรารวบรวม 10 วิธีจากโพสต์นั้น พร้อมเสริมข้อมูลจริงจาก Anthropic มาให้ครบ

## 1. Edit Prompt แทนส่งใหม่

ทุกครั้งที่คุณส่งข้อความใหม่ Claude ต้องอ่าน**ประวัติแชททั้งหมด**ตั้งแต่ต้น ยิ่งแชทยาว ยิ่งกิน token มหาศาล

**วิธีแก้:** ถ้า Claude ตอบไม่ตรง ให้กดปุ่ม **Edit** ที่ข้อความเดิมของคุณแล้ว regenerate — ไม่ต้องพิมพ์ใหม่ ไม่ต้องโหลดประวัติซ้ำ ประหยัดได้มาก

## 2. เปิดแชทใหม่ทุก 20 ข้อความ

Token cost เพิ่มขึ้น**ทุกข้อความ** เพราะ Claude ต้องอ่าน context ทั้งหมดก่อนตอบทุกครั้ง แชทยาว 50 ข้อความ = token cost ต่อข้อความสูงกว่าแชทใหม่หลายเท่า

**วิธีแก้:** พอแชทถึง 20 ข้อความ → ขอให้ Claude สรุปสิ่งที่คุยกันมา → copy สรุป → เปิดแชทใหม่วาง context ลงไป ได้คำตอบดีเท่าเดิม แต่ประหยัด token ลงครึ่ง

## 3. รวมคำถามไว้ในข้อความเดียว

3 คำถาม 3 ข้อความ = Claude โหลด context 3 รอบ

3 คำถามในข้อความเดียว = Claude โหลด context 1 รอบ

**วิธีแก้:** เตรียมทุกคำถามไว้ก่อน แล้วส่งทีเดียว ใช้เลขข้อหรือ bullet จัดระเบียบ Claude จะตอบทีละข้อให้ครบ

## 4. อัปโหลดไฟล์ลง Projects แทนแชท

ถ้าคุณใช้ไฟล์ PDF หรือเอกสารซ้ำบ่อยๆ อย่าอัปโหลดซ้ำทุกแชท

**วิธีแก้:** ใช้ฟีเจอร์ **Projects** ของ Claude — อัปโหลดไฟล์เข้า Project ครั้งเดียว ไฟล์จะถูก **cache** ไว้ ทุกแชทใน Project นั้นใช้ได้เลยโดยไม่ต้องโหลดซ้ำ Anthropic บอกว่า cache reads ประหยัดได้ถึง **90%** ของ token cost ปกติ

## 5. ตั้ง Memory และ User Preferences ให้เรียบร้อย

ทุกแชทใหม่ Claude เริ่มจากศูนย์ — ไม่รู้ว่าคุณเป็นใคร ทำอะไร ต้องการ output แบบไหน ถ้าต้องอธิบายซ้ำทุกครั้ง = เสีย token ฟรี

**วิธีแก้:** เข้า Settings > Memory and User Settings → บันทึกข้อมูลเกี่ยวกับตัวคุณ บทบาท สไตล์ที่ต้องการ Claude จะนำไปใช้อัตโนมัติทุกแชท

## 6. ปิดฟีเจอร์ที่ไม่ใช้

Web Search และ Extended Thinking เพิ่ม token ให้ทุกคำตอบ แม้คุณไม่ได้ต้องการ

- **Web Search:** Claude ค้นเว็บแล้วใส่ผลลัพธ์เข้ามา = token เพิ่ม
- **Extended Thinking:** Claude แสดงกระบวนการคิดทีละขั้น = thinking tokens เพิ่มมา (เริ่มต้น 1,024 tokens ขึ้นไป)

**วิธีแก้:** ปิด Search และ Extended Thinking ไว้เป็น default เปิดเฉพาะเมื่อจำเป็น

## 7. ใช้ Haiku สำหรับงานง่าย

ไม่ใช่ทุกงานต้องใช้ model ใหญ่ ตรวจแกรมมาร์ แปลภาษา สรุปข้อความ — Haiku ทำได้ดีในราคาถูกกว่า

| Model | ราคา Input | ราคา Output | เหมาะกับ |
|-------|-----------|-------------|---------|
| Opus 4.6 | $5/M | $25/M | งาน complex, coding ยาก |
| Sonnet 4.6 | $3/M | $15/M | งานทั่วไป (default) |
| Haiku 4.5 | $1/M | $5/M | งาน simple, draft, แปลภาษา |

Opus แพงกว่า Haiku **5 เท่า** — ถ้าใช้ Haiku สำหรับ 60% ของงาน จะประหยัดได้มหาศาล

## 8. กระจายการใช้งานตลอดวัน

Claude Pro ใช้ระบบ **rolling 5-hour window** — ไม่ได้ reset ตอนเที่ยงคืน

ถ้าคุณใช้หนักตอนเช้า 8-10 โมง แล้ว token หมด ต้องรอถึงบ่ายโมงกว่าจะกลับมา

**วิธีแก้:** แบ่งงานเป็น 2-3 session ตลอดวัน เช่น เช้า / บ่าย / ค่ำ — limit จะ refresh ทันรอบถัดไป

## 9. ใช้นอก Peak Hours

ช่วง **5:00-11:00 น. เวลา Pacific** (ตรงกับราว 19:00-01:00 เวลาไทย) เป็น peak hours ที่ limit หมดเร็วกว่าปกติ

**วิธีแก้:** ทำงานหนักช่วงเช้า-บ่ายเวลาไทย ซึ่งตรงกับ off-peak ของ Anthropic — ได้ใช้เต็มที่กว่า

## 10. เปิด Extra Usage เป็น Safety Net

ถ้า token หมดกลางทางงานสำคัญ Extra Usage จะให้คุณใช้ต่อได้ในอัตรา pay-as-you-go

**วิธีแก้:** เข้า Usage Settings → เปิด **Overage** → ตั้ง spending limit ไว้ (เช่น $10-20/เดือน) เพื่อป้องกันค่าใช้จ่ายเกิน

แต่ถ้าใช้ overage เกิน $80/เดือนเป็นประจำ → อัปเกรดเป็น Max 5x ($100/เดือน) ที่ได้ limit 5 เท่า + Opus access จะคุ้มกว่า

## สรุป: ประหยัด Token ไม่ใช่ใช้น้อยลง แต่ใช้ฉลาดขึ้น

| วิธี | ผลลัพธ์ |
|------|---------|
| Edit prompt แทนส่งใหม่ | ลด context reload |
| แชทใหม่ทุก 20 ข้อความ | ลด cost ต่อข้อความ |
| รวมคำถาม 1 ข้อความ | ลด context load 3x |
| ใช้ Projects + cache | ประหยัดได้ 90% |
| ตั้ง Memory ครั้งเดียว | ไม่ต้องอธิบายซ้ำ |
| ปิด Search/Thinking | ลด token ต่อคำตอบ |
| ใช้ Haiku งานง่าย | ถูกกว่า Opus 5 เท่า |
| กระจายตลอดวัน | limit refresh ทัน |
| ใช้นอก peak hours | ได้ limit เต็มที่ |
| เปิด Extra Usage | ไม่หยุดกลางงาน |

ไม่มีวิธีไหนที่ยากหรือต้องลงทุนเพิ่ม — แค่เปลี่ยนนิสัยการใช้งานเล็กน้อย ก็ได้ output จาก Claude มากขึ้นหลายเท่า
